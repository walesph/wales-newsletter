# WALES通信 자동화 시스템 — 기획서

> 매달 발행하는 WALES通信(뉴스레터)을 **웹으로 발행**하고, **AI로 전 언어 번역**해서
> **각국 유학 에이전트에게 자동 이메일 발송**하는 시스템.

---

## 1. 목표

| 현재 (수작업) | 자동화 후 |
|---|---|
| 佐藤 스태프가 일본어로 디자인(PDF) 제작 | 웹 폼에 내용 입력 → 자동 레이아웃 |
| 일본 에이전트에게만 일본어로 발송 | 전 언어(ja/ko/en/zh…) 에이전트에게 각 언어로 발송 |
| 매번 수동으로 메일 작성·첨부 | 버튼 한 번 / 또는 매월 예약 발송 |
| 번역은 안 함 (일본어만) | Claude API로 자동 번역 |

핵심 가치: **한 번 작성 → AI가 N개 언어로 번역 → 각국에 자동 발송 + 발송 이력 기록**

---

## 2. 뉴스레터 구성 (실제 Vol.17 기준)

PDF에서 확인한 섹션을 그대로 데이터화한다.

1. **헤더** — 발행일, Vol 호수, "WALES通信", 로고
2. **卒業生インタビュー (졸업생 인터뷰)**
   - 이름 / 성별 / 수강 기간(주) / 대표 인용구 / 본문
   - 인물 사진 또는 영상 썸네일
   - 인스타그램 핸들 + QR + "FOLLOW US"
3. **毎日バギオ情報 (SNS 안내)** — X(트위터) 링크/소개
4. **空き状況 (입학 가능 안내)** — 공실/만실 텍스트
5. **国籍·年齢·男女比率 (통계)**
   - 국적 비율 (도넛, %) — 대만/사우디/한국/일본/중국/러시아 등 **가변 국가 목록**
   - 남녀 비율 (도넛, %)
   - 연령대 비율 (막대, %)
6. **스태프 칼럼** — 자유 본문 + 사진 1~2장
7. **커버 인사말** — 이메일 본문 상단에 들어가는 담당자 개인 인사 (월별로 다름)

> 기존 스키마(`supabase_schema.sql`)에서 **보완할 필드**: `interview_gender`, `interview_weeks`,
> `instagram_handle`, `interview_video_url`, `sns_x_url`, `cover_message`, 그리고 통계는 이미 jsonb 라 국가 자유 확장 가능.

## 3. 다국어 발송 설계

- **마스터 언어 = 일본어**(佐藤가 작성). 이걸 원본으로 두고 나머지 언어는 AI 번역.
- 번역 대상 텍스트: 인터뷰 본문/인용구, 공실 안내, 스태프 칼럼, **커버 인사말**.
- 번역 **불필요**: 통계 숫자(언어 무관), 국가/성별/연령 **라벨**(코드 사전으로 번역 — 이미 i18n에 있음).
- 에이전트마다 `language` 필드 → 그 언어 버전 메일을 받음.

### 발송 언어 세트 (확정 — 7개)

| 코드 | 언어 | 비고 |
|---|---|---|
| `ja` | 일본어 | **마스터(원본)** — 번역 안 함 |
| `en` | 영어 | |
| `ko` | 한국어 | |
| `zh-Hant` | 중국어 번체 | 대만/홍콩 |
| `zh-Hans` | 중국어 간체 | 중국 |
| `vi` | 베트남어 | |
| `ar` | 아랍어 | **RTL** — 웹/PDF 모두 `dir="rtl"` 좌우반전 레이아웃 필요 |

→ 마스터(ja) 제외 **번역 대상 6개 언어**. 번체/간체는 별개 코드로 각각 번역(AI에 "번체로"/"간체로" 명시).

```
일본어 마스터 입력
      │
      ├─ Claude 번역 → ko 저장
      ├─ Claude 번역 → en 저장   →  newsletter_translations 테이블
      ├─ Claude 번역 → zh 저장
      │
발송: 각 에이전트.language 에 맞는 번역본으로 이메일 렌더 → Resend 발송 → send_logs 기록
```

## 4. 월간 운영 워크플로우

1. **작성** — 관리자 웹에서 이번 호 입력(인터뷰/통계/칼럼/커버 인사말), 사진 업로드.
2. **번역** — "전 언어 번역" 버튼 → Worker가 Claude API 호출 → 번역본 저장. (관리자가 검수·수정 가능)
3. **미리보기** — 언어 탭으로 각 언어 최종본 확인. 본인에게 **테스트 발송**.
4. **발행** — status = published (웹 공개 URL 생성).
5. **발송** — "전 에이전트 발송" 버튼 **또는** 매월 1일 Cron 자동 발송(옵션).
6. **이력** — 누구에게 언제 성공/실패했는지 `send_logs` 에 기록, 화면에서 확인.

## 5. 시스템 아키텍처

```
[관리자/구독자 브라우저]
        │
        ▼
[Cloudflare Pages]  ← React 앱 (작성·미리보기·발송 트리거, 웹 공개본)
        │ REST/JWT
        ▼
[Supabase]  Postgres(뉴스레터/에이전트/번역/발송로그) + RLS + 이미지 Storage
        ▲
        │ service_role
[Cloudflare Worker]  ── POST /send      이메일 일괄/테스트 발송 (HTML 본문 + PDF 첨부)
        │            ── POST /translate AI 번역 (Claude, 6개 언어)
        │            ── POST /pdf       언어별 PDF 생성 (Browser Rendering)
        ├──────► [Resend]            이메일 전송 (walesportal.com 도메인)
        ├──────► [Anthropic]         ja → en/ko/zh-Hant/zh-Hans/vi/ar 번역
        └──────► [Browser Rendering] 웹 뉴스레터 → PDF 렌더
```

- **발송 = 수동 버튼만** (Cron 예약 미사용 — 결정됨). Worker에 `scheduled` 핸들러는 비활성/제거.
- **이메일 = HTML 본문 + 언어별 PDF 첨부 둘 다** (결정됨). PDF는 웹 뷰를 Browser Rendering으로 렌더해 재사용.
- **번역 검수 필수** — AI 번역 → 관리자 확인/수정 → 발송 (결정됨).

- **GitHub** = 소스 + Actions(Worker 자동배포). Pages는 저장소 연결로 자동배포.
- 이미 구축됨: 저장소, Worker(send), 스키마, 프론트 골격. **추가 필요**: /translate, 관리자 UI, 보완 필드.

## 6. AI 번역 설계 (비용 직결)

- 한 호의 번역 대상 텍스트량 ≈ 일본어 2,000~3,000자 (대략 2,000~3,500 토큰).
- 언어당 1회 호출, 3개 언어(ko/en/zh) → 월 3~4회 호출.
- **모델 선택이 비용을 좌우** → 번역은 Haiku/Sonnet로 충분(아래 비용표 참고).
- JSON 구조로 입력/출력(필드별 번역) → 파싱 안정적.

## 7. 💰 비용 분석 (핵심)

> 결론부터: **AI 번역은 비용의 거의 0%**(월 몇 백 원). 실질 변수는 **수신 에이전트 수**(Resend)와
> Workers 유료 전환(월 $5) 정도. 규모별로 **월 $0 ~ $25** 수준.

가정: 월 1회 발행, 호당 번역 텍스트 ≈ 3,000 토큰, 3개 언어 번역.

### 7-1. 서비스별 (2026년 기준, 정책은 변동 가능 — 배포 전 확인)

| 서비스 | 무료 한도 | 유료 시작 | 우리 사용량 | 판단 |
|---|---|---|---|---|
| **Cloudflare Workers** | 10만 req/일, CPU 10ms, 서브요청 50개 | **$5/월** (서브요청 1,000, CPU 5분) | 월 몇 회 발송, 수신자만큼 서브요청 | 수신자 ≤ 수십이면 무료 가능, **안정성 위해 $5 권장** |
| **Cloudflare Pages** | 무제한 요청, 빌드 500/월 | — | 정적 호스팅 | **$0** |
| **Supabase** | DB 500MB, Storage 1GB | $25/월(Pro) | 데이터 수 MB | **$0** |
| **Resend** | 3,000통/월, **100통/일** | **$20/월** (5만통, 일일제한 완화) | 200통/월 | 한 번에 발송하려면 **$20** (또는 2~3일 분할 시 무료) |
| **Browser Rendering (PDF)** | Workers 유료에 포함(소량 무료) | 사용량 종량 | 월 7개 PDF(언어당 1개) | 사실상 **$0** |
| **Anthropic (번역)** | — (종량제) | 토큰당 | 월 6언어 번역, 소량 | **월 $0.3~1** |

### 7-2. AI 번역 실제 토큰 비용 (월)

입력 3,000토큰 × 3언어 = 9,000토큰, 출력 ≈ 9,000토큰 가정:

| 모델 | 입력/출력 단가(대략, /M토큰) | 월 비용(번역) |
|---|---|---|
| Haiku 4.5 | ~$1 / ~$5 | **≈ $0.05** |
| Sonnet 4.6 | ~$3 / ~$15 | **≈ $0.16** |
| Opus 4.8 | ~$15 / ~$75 | **≈ $0.8** |

→ 번역은 **Sonnet 권장**(품질·비용 균형). 그래도 **월 20센트 수준**. 비용 걱정 대상 아님.

### 7-3. 우리 케이스 확정 (에이전트 200명 · 7언어 · PDF+HTML)

| 항목 | 선택 | 월 비용 |
|---|---|---|
| Cloudflare Workers | 유료(안정) | $5 |
| Browser Rendering(PDF) | Workers에 포함 | ≈ $0 |
| Anthropic 번역 | Sonnet, 6언어 | ≈ $0.3 |
| Supabase / Pages | 무료 | $0 |
| **Resend (200통/월)** | **A. Pro 한방 발송** | **$20** |
| | **B. 무료 + 2~3일 분할(하루 100통)** | **$0** |

- **A안 (한 번 클릭 = 200통 즉시):** 합계 **≈ $25/월 (약 3.5만원)**
- **B안 (무료 유지, 분할 발송):** 합계 **≈ $5/월 (약 7천원)** — 단, 발송을 2~3일에 나눠 눌러야 함

### 7-4. 결론
- **AI 번역 때문에 비용 폭증? → 아니오.** 6개 언어로 늘어도 **월 몇 백 원**.
- PDF 자동 생성도 추가비 거의 없음(Workers 유료에 포함).
- **200명 기준 현실 운영비: 월 $5(분할) ~ $25(한방).** → **Resend만 결정하면 끝.**

## 8. 구현 로드맵

- **Phase 0 (완료)** — 저장소/프론트 골격/스키마/Worker(send)/CI.
- **Phase 1 — 데이터 모델 보완**: 인터뷰 성별·기간·인스타, 커버 인사말, SNS 링크, 이미지 Storage.
- **Phase 2 — 디자인 정합**: 실제 PDF 레이아웃에 맞춰 웹/이메일 템플릿(도넛 차트, 2단 구성, 사진).
- **Phase 3 — AI 번역**: Worker `/translate` 엔드포인트 + 관리자 "전 언어 번역/검수" UI.
- **Phase 4 — 관리자 UI**: 작성·미리보기·테스트발송·발행·일괄발송·발송이력.
- **Phase 5 — 배포/운영**: Cloudflare Pages 연결, Resend 도메인 인증, 시크릿 등록, (옵션)월간 Cron.

## 9. 결정 사항 (확정)

1. **발송 언어** — ✅ 7개: ja(마스터)/en/ko/zh-Hant/zh-Hans/vi/ar. (번역 6개, ar은 RTL)
2. **이메일 형식** — ✅ **HTML 본문 + 언어별 PDF 첨부 둘 다 유지**.
3. **발송 방식** — ✅ **수동 버튼만** (Cron 자동 발송 미사용).
4. **번역 검수** — ✅ **관리자 확인 후 발송** (AI 번역 → 검수 → 발송).
5. **에이전트 규모** — ✅ 약 200명.
6. **이미지 호스팅** — Supabase Storage 사용(권장, 기본 채택).

7. **Resend** — ✅ **무료 + 분할 발송** ($0). 하루 100통 제한 준수.

### 분할 발송 로직 요건 (무료 유지용)
- 발송 버튼은 **아직 안 보낸 에이전트에게만** 발송 (이미 보낸 사람은 `send_logs`로 자동 제외 — 유니크 인덱스가 보장).
- **1회 실행당 상한(예: 90~100통)** 적용 → 하루 한도 초과 방지.
- 화면에 **"전체 200 / 발송 완료 X / 남은 Y"** 진행률 표시 → 다음 날 다시 눌러 마저 발송.
- 즉 200명 = **약 2~3일에 걸쳐 버튼 클릭**으로 완료. (월 1회 발행이라 부담 적음)

### 운영비 확정: **월 약 $5 (약 7천원)** — Workers $5 + 번역 $0.3 + 나머지 무료.
